    Client¶

    CryptoVol ¶

    Synchronous client for the CryptoVol REST API.

    Parameters:

    Name Type Description Default
    api_key str

    Your RapidAPI key. Get one at https://www.cryptovol.io/api. Sent as X-RapidAPI-Key on every request.

    		 required
    base_url str

    Override the API base URL. Defaults to the RapidAPI gateway.

    		 DEFAULT_BASE_URL
    host str

    Value for the X-RapidAPI-Host header. You won't need to change this unless you're routing through a different gateway.

    		 DEFAULT_HOST
    timeout float

    Per-request timeout in seconds. Defaults to 30s.

    		 DEFAULT_TIMEOUT
    max_retries int

    How many times to retry transient failures (network errors and 5xx responses) with exponential backoff. Set to 0 to disable.

    		 DEFAULT_MAX_RETRIES
    user_agent str

    Custom User-Agent header. Useful if you're embedding the SDK in your own product and want clean attribution in server logs.

    		 DEFAULT_USER_AGENT

    vol_surface ¶

    vol_surface(ccy: str, expiry: str, strike_type: StrikeType, strike_value: float, *, option_type: OptionType = 'C', session: Session = 'us', date: Optional[str] = None, include_analytics: bool = False, raw: bool = False) -> Union[VolSurfacePoint, Dict[str, Any]]

    Single-point SABR-interpolated implied vol query.

    Parameters:

    Name Type Description Default
    ccy str

    Asset symbol.

    		 required
    expiry str

    Option expiry date, YYYY-MM-DD. Must be in the future relative to the snapshot date.

    		 required
    strike_type StrikeType

    How strike_value is interpreted:

    • "strike" — absolute strike price (e.g. 100_000)
    • "moneyness" — K/S ratio (e.g. 1.0 for ATM, 0.9 for 10% OTM put)
    • "delta" — signed/unsigned BS delta magnitude (e.g. 0.25)
    		 required
    strike_value float

    The numeric strike, moneyness, or delta — see strike_type.

    		 required
    option_type OptionType

    "C" (call, default) or "P" (put). Required when strike_type="delta"; ignored otherwise.

    		 'C'
    session Session

    "asia", "london", or "us". Subject to your plan tier.

    		 'us'
    date Optional[str]

    Optional snapshot date (YYYY-MM-DD). Defaults to the latest available snapshot.

    		 None
    include_analytics bool

    If True, also return spot, forward, yield rate, BS price, and Greeks. Requires a PRO or ULTRA plan.

    		 False
    raw bool

    If True, return the parsed JSON dict instead of a typed model.

    		 False

    vol_surface_bulk ¶

    vol_surface_bulk(ccy: str, queries: List[Dict[str, Any]], *, session: Session = 'us', date: Optional[str] = None, raw: bool = False) -> Union[BulkVolResponse, Dict[str, Any]]

    Resolve many (expiry, strike) points in a single request.

    Far more efficient than looping over :meth:vol_surface — one snapshot is loaded once and reused for every query.

    Parameters:

    Name Type Description Default
    ccy str

    Asset symbol.

    		 required
    queries List[Dict[str, Any]]

    List of dicts, each with keys:

    • expiry (str, required) — YYYY-MM-DD
    • strike_type (str, required) — "strike", "moneyness", or "delta"
    • strike_value (float, required)
    • option_type (str, optional) — "C" or "P" (default "C")
    • include_analytics (bool, optional) — default False
    		 required
    session Session

    "asia", "london", or "us".

    		 'us'
    date Optional[str]

    Optional snapshot date.

    		 None
    raw bool

    If True, return the parsed JSON dict instead of a typed model.

    		 False
    Example 
    response = cv.vol_surface_bulk(
    ccy="BTC",
    queries=[
        {"expiry": "2026-09-26", "strike_type": "moneyness", "strike_value": 0.9},
        {"expiry": "2026-09-26", "strike_type": "moneyness", "strike_value": 1.0},
        {"expiry": "2026-09-26", "strike_type": "moneyness", "strike_value": 1.1},
        {"expiry": "2026-09-26", "strike_type": "delta",
         "strike_value": 0.25, "option_type": "C", "include_analytics": True},
    ],
)
for item in response.successful:
    print(item.expiry, item.strike, item.vol)

    vol_history ¶

    vol_history(ccy: str = 'BTC', tenor: str = '1M', strike_type: Literal['moneyness', 'delta'] = 'moneyness', strike_value: float = 1.0, *, option_type: OptionType = 'C', start_date: Optional[str] = None, end_date: Optional[str] = None, raw: bool = False) -> Union[VolHistoryResponse, Dict[str, Any]]

    Historical time series at a constant maturity and constant strike.

    Ideal for backtesting, regression analysis, and vol regime monitoring.

    Requires a PRO or ULTRA plan (BASIC plans get 403 on this endpoint).

    Parameters:

    Name Type Description Default
    ccy str

    Asset symbol.

    		 'BTC'
    tenor str

    Constant-maturity tenor: 1D, 1W, 2W, 3W, 1M, 2M, 3M, 6M, 9M, or 1Y.

    		 '1M'
    strike_type Literal['moneyness', 'delta']

    "moneyness" (K/S) or "delta". Absolute strikes are not supported for history queries.

    		 'moneyness'
    strike_value float

    Must be a value on the published grid — see the API docs for the full list. Common moneyness points: 0.9, 0.95, 1.0, 1.05, 1.1. Common deltas: 0.10, 0.25.

    		 1.0
    option_type OptionType

    "C" or "P". Required when strike_type="delta".

    		 'C'
    start_date Optional[str]

    YYYY-MM-DD bounds. PRO plans are limited to the last year.

    		 None
    raw bool

    If True, return the parsed JSON dict instead of a typed model.

    		 False

    Exceptions¶

    CryptoVolError ¶

    Bases: Exception

    Base exception for all errors raised by the CryptoVol SDK.

    AuthenticationError ¶

    Bases: CryptoVolError

    Raised on 401 / 403 — invalid or missing API key, or non-allowed origin.

    PlanLimitError ¶

    Bases: CryptoVolError

    Raised on 403 when the request hits a plan-tier limit.

    Examples: querying ETH on a BASIC plan, requesting Greeks without PRO, querying dates older than your plan's history window.

    The server's error message typically tells you which tier unlocks the feature.

    NotFoundError ¶

    Bases: CryptoVolError

    Raised on 404 — no data available for the requested asset/date/session.

    ValidationError ¶

    Bases: CryptoVolError

    Raised on 400 / 422 — malformed parameters (bad ccy, bad date format, etc.).

    RateLimitError ¶

    Bases: CryptoVolError

    Raised on 429 — daily/monthly quota exceeded.

    ServerError ¶

    Bases: CryptoVolError

    Raised on 5xx — transient server-side failure.

    TimeoutError ¶

    Bases: CryptoVolError

    Raised when a request exceeds the configured timeout.
